## Critical Gotchas

### 1. Docker Secrets Permission Issue
The most impactful bug: Docker Compose file-based secrets mount files with `ubuntu:ubuntu` ownership and `640` permissions. The webapp runs Tomcat as `tomcat_admin`, which cannot read the secret file. **Fix**: Use a regular volume mount to `/run/secrets/common.properties` instead of Docker secrets:
```yaml
volumes:
  - ./volume/properties/common.properties:/run/secrets/common.properties:ro
```

### 2. Docker Compose v2 Package
The package name varies by Ubuntu version:
- Ubuntu 22.04: `docker-compose-plugin` or `docker-compose-v2`
- Fallback: `docker-compose` (v1, uses `docker-compose` instead of `docker compose`)

### 3. Snap Firefox Profile Paths
Firefox on Ubuntu 22.04 is a snap. The profile must exist in **both** locations:
- `/home/ga/.mozilla/firefox/default-release/` (standard)
- `/home/ga/snap/firefox/common/.mozilla/firefox/default-release/` (snap)

The `--profile` flag does NOT work reliably with snap Firefox. Instead, use `profiles.ini` with `StartWithLastProfile=1`.

### 4. React Login Automation
Direct xdotool typing into React form fields does NOT update React state (the `onChange` handler never fires). The proven workaround:
```javascript
fetch('/api/OpenELIS-Global/ValidateLogin', {
  method: 'POST',
  headers: {'Content-Type': 'application/x-www-form-urlencoded'},
  body: 'loginName=admin&password=adminADMIN!',
  credentials: 'include'
}).then(r => { window.location.href = '/'; });
```

### 5. SSL Certificate Handling (Multi-Layer)
Self-signed cert generated by `itechuw/certgen:main` container. Four-layer approach:
1. System CA store via `update-ca-certificates`
2. Firefox NSS database via `certutil -A` into all profile directories
3. Firefox Enterprise Policies (`policies.json`) with `ImportEnterpriseRoots`
4. `cert_override.txt` with SHA-256 fingerprint for localhost:443
5. Mouse-click fallback: Advanced → Accept the Risk (coordinates at 1920x1080: 1319,705 then 1319,800)

### 6. Carbon Design System Sidebar
The sidebar navigation uses IBM Carbon's `SideNav`. The menu toggle button must be clicked via JavaScript:
```javascript
document.querySelector('.cds--header__menu-trigger').click();
```

### 7. URL Routing
The React frontend serves at `https://localhost/` while the backend API is at `https://localhost/api/OpenELIS-Global/`. Direct navigation to API URLs (e.g., `/api/OpenELIS-Global/SampleAdd`) returns JSON errors.

### 8. Docker Config Permissions
`docker login` in pre_start runs as root, creating `/home/ga/.docker/config.json` owned by root. Must `chown -R ga:ga /home/ga/.docker` in post_start.

### 9. Login Endpoint
- **React frontend login**: `POST /api/OpenELIS-Global/ValidateLogin` with form-urlencoded `loginName=admin&password=adminADMIN!`
- **Session check**: `GET /api/OpenELIS-Global/session` returns `{"authenticated":true,...}`
- **Spring Security**: `j_spring_security_check` returns 403 (CSRF required) — do NOT use
